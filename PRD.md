## 一、产品目标（Goal）

> **让没有系统学过AI的人，在30天内建立完整的AI概念框架，能看懂行业讨论、能做工具选型判断、能开始动手实践。**

**可衡量的成功指标：**

| 指标 | MVP目标 | 6个月目标 |
|------|---------|----------|
| 完成一个完整模块的用户比例 | 40% | 60% |
| 完成作业并提交GitHub的用户数 | 50人 | 500人 |
| 用户7日留存率 | 25% | 40% |
| NPS（净推荐值） | >30 | >50 |

## 二、问题定义

**现有AI学习资源的三个缺口：**

1. **太浅** — 公众号科普停留在"是什么"，没有"为什么"和"怎么用"
2. **太深** — 论文和英文课程门槛高，非技术背景用户放弃
3. **碎片** — 概念散落各处，没有系统化的学习路径和关联图

**这个产品填补中间地带：** 有技术深度，但用人话讲；系统完整，但学习单元足够小。

---

## 三、目标用户

**主要用户：**  
- 开发者社区成员，有一定编程背景，但没系统学过AI/ML
- 想转型做AI产品的人，需要建立概念体系

**次要用户：**  
- 非技术背景但对AI感兴趣的产品经理/设计师
- 正在使用Claude Code、Cursor等工具，想更深理解背后原理的工程师

**不是目标用户：**  
- 想学训练模型的ML研究员（太深）
- 完全不接触AI的大众（太远）

## 五、核心功能

### F1：概念卡片系统
- 每张卡片包含：简单解释 / 深入理解 / 真实场景 / 关联概念
- 支持按模块浏览、关键词搜索、难度筛选
- 「我学会了」标记 + 进度存本地

### F2：间隔重复系统
- 学完的概念进入复习队列
- 答对 → 间隔延长（3天 → 7天 → 14天）
- 答错 → 重新出现，附更详细解释
- 进度显示：「稳定记住了 X 个概念」而非「读了X张卡片」

### F3：交互演示
- Temperature滑块：预设多组输出，滑动对比
- Prompt对比：好/坏prompt的输出差异并排展示
- RAG流程动画：检索→增强→生成逐步可视化
- CoT对比：有/没有思维链的推理过程

### F4：作业系统（GitHub集成）
- 每个概念配一道具体作业（15-30分钟完成）
- 作业模板存储在 GitHub repo：`ai-concepts-homework`
- 用户流程：Fork → 完成 → Commit → 可选提交PR
- 完成后卡片标注「已实践」徽章

### F5：模块测验
- 每模块末尾 5-8 道题
- 题型：单选（定义理解）+ 场景判断（应用能力）
- 答错显示详细解释
- 通过测验才能解锁下一模块（可选，用户可跳过）

## 六、用户旅程

```
进入首页
   ↓
看到11个模块 + 整体进度
   ↓
选择一个模块（建议从模块01开始）
   ↓
浏览模块内概念列表
   ↓
点开一个概念 → 读简单解释 → 展开深入理解 → 看真实场景
   ↓
点「我学会了」→ 概念进入复习队列
   ↓
做作业（跳转GitHub模板）→ 完成后标记
   ↓
全部概念学完 → 做模块测验
   ↓
进入下一模块 / 接到复习提醒
```

## 七、技术方案

**MVP阶段：**
- 纯前端：HTML + CSS + JavaScript（无需后端）
- 进度存储：localStorage
- 作业系统：GitHub repo（外链）
- 部署：GitHub Pages / Vercel（静态托管，免费）

**后期可扩展：**
- 添加真实API调用（用户输入自己的API Key）
- 社区功能：看到别人的作业提交
- 概念关系图可视化

## 八、不做什么（Out of Scope）

- ❌ 用户账号系统（MVP阶段）
- ❌ 付费墙或课程销售
- ❌ 视频内容
- ❌ 真实AI API调用（MVP交互演示用预设数据）
- ❌ 移动App
- ❌ 多语言（英文版）
